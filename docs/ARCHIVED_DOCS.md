Co dalej?

1. Przegląd: przejrzeć każdy plik i oznaczyć go "keep" / "update" / "archive" / "delete".
2. Archiwizacja: możliwe działania dla plików oznaczonych jako `archive`:
   - Dodać nagłówek `ARCHIWALNE — NIEAKTUALNE` i przenieść oryginał do `docs/archive/` albo
   - Zachować plik w miejscu i dodać baner wskazujący, że jest przestarzały.
3. Usuwanie: usunąć pliki tylko po zatwierdzeniu zespołu i wprowadzeniu kopii zapasowej (np. branch `archive-cleanup`).

Chcesz, żebym automatycznie oznaczył powyższe pliki banerem "ARCHIWALNE" i/lub przeniósł ich treść do `docs/archive/`? Jeśli tak, wybierz preferowaną metodę (baner vs. przeniesienie) — mogę zrobić to teraz w jednym kroku.
